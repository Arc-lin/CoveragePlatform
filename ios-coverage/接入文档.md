## 1. 方案概述

iOS 端采用 **LLVM Sanitizer Coverage** 进行代码覆盖率统计，配合 Git Diff 实现增量覆盖率分析。

### 核心原理

```
编译时插桩            运行时回调              数据导出             报告生成
(Clang flags)  →  (__sanitizer_cov_*)  →  (.profraw)  →  (lcov/HTML)
                                                              ↓
                                          CI 流程：Git Diff → diffParser.rb → 增量报告
```

| 阶段 | 技术 | 说明 |
|------|------|------|
| 编译时插桩 | `-fprofile-instr-generate -fcoverage-mapping` | Clang 编译器在代码中插入探针 |
| 运行时回调 | `__sanitizer_cov_trace_pc_guard` | 程序执行时记录代码覆盖 |
| 数据导出 | `__llvm_profile_write_file()` | 导出 `.profraw` 原始覆盖率数据 |
| 数据合并 | `llvm-profdata merge` | 将多个 `.profraw` 合并为 `.profdata` |
| 报告生成 | `llvm-cov export` + `genhtml` | 导出 lcov 格式并生成 HTML 报告 |

### 与 Android 方案对比

| 特性 | iOS | Android |
|------|-----|---------|
| 插桩工具 | LLVM Sanitizer Coverage / Clang | JaCoCo |
| 数据格式 | `.profraw` → `.profdata` | `.ec` |
| 报告工具 | `llvm-cov` + `genhtml` | JaCoCo XML |
| 上传方式 | Shell 脚本 / cURL | OkHttp 内嵌上传 |
| 增量分析 | Ruby `diffParser.rb` + `genhtml` | Python `incremental_coverage.py` |

### 文件清单

| 文件 | 用途 |
|------|------|
| `CoverageCollector.h` | Objective-C 头文件（对外 API） |
| `CoverageCollector.m` | Objective-C 实现（LLVM Profile 调用 + Sanitizer Coverage 回调） |
| `CoverageCollector.swift` | Swift 版本实现（可选） |
| `coverage_report.sh` | Shell 脚本：一键生成全量 / 增量覆盖率报告 |
| `gitdiff/utils/diffParser.rb` | Ruby 脚本：增量覆盖率解析（Git Diff → lcov gather） |

## 2. 快速开始

### 三步集成

**Step 1：添加编译参数**

在 Xcode Build Settings 中添加：

```
Other C Flags:    -fprofile-instr-generate -fcoverage-mapping
Other Swift Flags: -sanitize-coverage=func -sanitize=undefined
Other Linker Flags: -fprofile-instr-generate
```

**Step 2：添加源文件**

将 `CoverageCollector.h` 和 `CoverageCollector.m` 拖入 Xcode 项目。

**Step 3：AppDelegate 中初始化**

```objc
// AppDelegate.m
#import "CoverageCollector.h"

- (BOOL)application:(UIApplication *)application 
    didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    // 初始化覆盖率收集器
    [CoverageCollector initializeCollector];
    return YES;
}

- (void)applicationDidEnterBackground:(UIApplication *)application {
    // App 进入后台时保存覆盖率数据
    [CoverageCollector dumpCoverageData];
}

- (void)applicationWillTerminate:(UIApplication *)application {
    // App 即将终止时保存覆盖率数据
    [CoverageCollector dumpCoverageData];
}
```

完成以上三步后，运行 App 即可自动收集覆盖率数据。

## 3. Xcode 编译配置

### 3.1 Objective-C / C / C++ 文件

在 **Build Settings** 中搜索并配置以下项：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **Other C Flags** | `-fprofile-instr-generate -fcoverage-mapping` | 启用 Clang 代码覆盖率插桩 |
| **Other Linker Flags** | `-fprofile-instr-generate` | 链接覆盖率运行时库 |

**方式一（推荐）：Clang Source-based Coverage**

```
-fprofile-instr-generate -fcoverage-mapping
```

> 这是 Apple 推荐的覆盖率方案，配合 `llvm-cov` 工具链使用，兼容性好。

**方式二：Sanitizer Coverage**

```
-sanitize-coverage=func,trace-pc-guard
```

> 更底层的插桩方式，通过 `__sanitizer_cov_trace_pc_guard` 回调精确控制。适合需要自定义回调逻辑的场景。

### 3.2 Swift 文件

在 **Build Settings** 中搜索并配置：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **Other Swift Flags** | `-sanitize-coverage=func -sanitize=undefined` | Swift 覆盖率插桩 |

### 3.3 仅 Debug 配置启用（推荐）

为避免 Release 包体积增大和性能影响，建议仅在 Debug 配置下启用：

1. 在 Xcode 中选择 **Build Settings**
2. 展开配置项左侧的三角箭头
3. 仅在 **Debug** 行填写上述编译参数，**Release** 行留空

### 3.4 配置截图参考

```
Build Settings:
├── Other C Flags
│   ├── Debug:   -fprofile-instr-generate -fcoverage-mapping
│   └── Release: (空)
├── Other Swift Flags
│   ├── Debug:   -sanitize-coverage=func -sanitize=undefined
│   └── Release: (空)
└── Other Linker Flags
    ├── Debug:   -fprofile-instr-generate
    └── Release: (空)
```

### 3.5 验证编译参数是否生效

编译后在 Xcode Build Log 中搜索 `-fprofile-instr-generate`，确认编译命令中包含该参数。

## 4. SDK 集成（Objective-C）

### 4.1 文件引入

将以下文件添加到 Xcode 项目中：

- `CoverageCollector.h`
- `CoverageCollector.m`

> ⚠️ 注意：`CoverageCollector.m` 中包含 `Sanitizer Coverage` 回调函数（`__sanitizer_cov_trace_pc_guard_init` / `__sanitizer_cov_trace_pc_guard`），**不要**在同一个 Target 中重复定义这些函数。

### 4.2 头文件 API

```objc
// CoverageCollector.h
@interface CoverageCollector : NSObject

/// 初始化覆盖率收集器，设置输出文件路径
/// 建议在 Application 启动时调用
+ (void)initializeCollector;

/// 手动触发覆盖率数据保存
/// @return 是否保存成功
+ (BOOL)dumpCoverageData;

/// 获取覆盖率数据文件路径
+ (nullable NSString *)coverageFilePath;

/// 获取所有历史覆盖率文件
+ (NSArray<NSString *> *)allCoverageFiles;

/// 清除所有覆盖率数据
+ (void)clearCoverageData;

/// 设置覆盖率数据输出目录（可选，默认为 Document/Coverage 目录）
+ (void)setCoverageDirectory:(NSString *)directory;

@end
```

### 4.3 实现原理

`CoverageCollector.m` 核心实现包括两部分：

**① Sanitizer Coverage 回调**（编译时自动调用）

```objc
// 初始化回调：为每个探针分配唯一 ID
void __sanitizer_cov_trace_pc_guard_init(uint32_t *start, uint32_t *stop) {
    static uint64_t N;
    if (start == stop || *start) return;
    for (uint32_t *x = start; x < stop; x++) {
        *x = ++N;
    }
}

// 执行回调：代码执行到探针点时触发
void __sanitizer_cov_trace_pc_guard(uint32_t *guard) {
    void *PC = __builtin_return_address(0);
    Dl_info info;
    dladdr(PC, &info);
    // 可在此处添加自定义日志
}
```

**② LLVM Profile 文件操作**（手动调用）

```objc
// 设置输出文件路径
__llvm_profile_set_filename([coveragePath UTF8String]);

// 写入覆盖率数据到文件
int result = __llvm_profile_write_file();
// result == 0 表示成功
```

### 4.4 自定义输出目录

```objc
// 设置自定义输出目录
NSString *customDir = [NSTemporaryDirectory() stringByAppendingPathComponent:@"MyCoverage"];
[CoverageCollector setCoverageDirectory:customDir];
```

默认输出目录为 `Documents/Coverage/`。

## 5. SDK 集成（Swift）

### 5.1 文件引入

将 `CoverageCollector.swift` 添加到 Xcode 项目中。

### 5.2 Swift API

```swift
public class CoverageCollector {

    /// 初始化覆盖率收集器
    public static func initialize()

    /// 保存覆盖率数据
    /// @return 是否保存成功
    @discardableResult
    public static func dumpCoverageData() -> Bool

    /// 获取覆盖率文件路径
    public static func coverageFilePath() -> URL?

    /// 获取所有覆盖率文件
    public static func allCoverageFiles() -> [URL]

    /// 清除所有覆盖率数据
    public static func clearCoverageData()

    /// 设置覆盖率输出目录
    public static func setCoverageDirectory(_ directory: URL)
}
```

### 5.3 使用示例

```swift
// AppDelegate.swift
import UIKit

@main
class AppDelegate: UIResponder, UIApplicationDelegate {

    func application(_ application: UIApplication,
                     didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        // 初始化
        CoverageCollector.initialize()
        return true
    }

    func applicationDidEnterBackground(_ application: UIApplication) {
        // 进入后台时保存
        CoverageCollector.dumpCoverageData()
    }

    func applicationWillTerminate(_ application: UIApplication) {
        // 即将终止时保存
        CoverageCollector.dumpCoverageData()
    }
}
```

### 5.4 Swift 实现原理

Swift 版本通过 `dlopen` + `dlsym` 动态查找 LLVM Profile 函数：

```swift
private static let llvmProfileWriteFile: @convention(c) () -> Int32 = {
    let handle = dlopen(nil, RTLD_NOW)
    let sym = dlsym(handle, "__llvm_profile_write_file")
    return unsafeBitCast(sym, to: (@convention(c) () -> Int32).self)
}()
```

> ⚠️ Swift 版本使用 `DispatchQueue.once` 扩展来保证初始化只执行一次，确保 `__llvm_profile_set_filename` 不会重复调用。

## 7. 覆盖率数据提取

### 7.1 数据文件说明

| 格式 | 说明 |
|------|------|
| `.profraw` | LLVM Profile 原始数据（App 运行时生成） |
| `.profdata` | 合并/索引后的数据（`llvm-profdata merge` 生成） |

**文件命名规则：**

```
<bundle_identifier>_<yyyyMMdd_HHmmss>.profraw
```

示例：`com.example.app_20260615_140530.profraw`

### 7.2 从真机提取覆盖率文件

**方式一：Xcode Devices Window**

1. 打开 Xcode → Window → Devices and Simulators
2. 选择你的设备
3. 找到你的 App → 点击齿轮图标 → Download Container
4. 解压 `.xcappdata` 文件
5. 进入 `AppData/Documents/Coverage/` 目录找到 `.profraw` 文件

**方式二：通过 iTunes / Finder 文件共享**

如果 App 启用了文件共享（`UIFileSharingEnabled`），可以通过 Finder 直接访问 Documents 目录。

**方式三：libimobiledevice（命令行）**

```bash
# 安装 libimobiledevice
brew install libimobiledevice

# 挂载开发者磁盘镜像
ideviceimagemounter /path/to/DeveloperDiskImage.dmg

# 列出 App 的 Documents 目录
idevicefs ls /Documents/Coverage/

# 下载文件
idevicefs pull /Documents/Coverage/ ~/Desktop/coverage/
```

### 7.3 从模拟器提取

```bash
# 模拟器数据目录
cd ~/Library/Developer/CoreSimulator/Devices/

# 查找你的 App
find . -name "*.profraw" -path "*/Documents/Coverage/*"

# 复制到桌面
cp <path_to_profraw> ~/Desktop/
```

### 7.4 编程方式获取

```objc
// 获取当前覆盖率文件路径
NSString *path = [CoverageCollector coverageFilePath];

// 获取所有历史文件
NSArray<NSString *> *allFiles = [CoverageCollector allCoverageFiles];
for (NSString *file in allFiles) {
    NSLog(@"覆盖率文件: %@", file);
}

// 清除旧数据
[CoverageCollector clearCoverageData];
```

## 8. 覆盖率报告生成

### 8.1 工具链说明

| 工具 | 用途 |
|------|------|
| `llvm-profdata` | 将 `.profraw` 转换为 `.profdata`（索引/合并） |
| `llvm-cov` | 从 `.profdata` 导出 lcov 格式覆盖率数据 |
| `genhtml` | 将 lcov 数据转换为 HTML 可视化报告 |
| `coverage_report.sh` | 一键脚本，封装以上所有步骤 |

### 8.2 前置条件

确保安装了必要工具：

```bash
# llvm-profdata / llvm-cov（随 Xcode 安装）
xcrun llvm-profdata --version
xcrun llvm-cov --version

# genhtml（通过 Homebrew 安装）
brew install lcov
```

### 8.3 准备二进制文件

覆盖率报告需要 **带符号的 Mach-O 二进制文件** 才能解析。准备方式：

**方式一：使用 .app 包中的二进制文件**

```bash
# 创建 MachOFiles 目录
mkdir -p ./MachOFiles

# 复制 .app 包中的二进制文件
cp -R /path/to/YourApp.app ./MachOFiles/
```

**方式二：从 Xcode Archive 中提取**

```bash
# Xcode → Product → Archive
# 导出 .app 后复制到 MachOFiles/
mkdir -p ./MachOFiles
cp -R /path/to/exported/YourApp.app ./MachOFiles/
```

> ⚠️ 二进制文件必须与 `.profraw` 文件来自**同一次构建**，否则报告会解析失败。

### 8.4 使用 coverage_report.sh 一键生成

```bash
# 语法
sh coverage_report.sh <profraw文件> [对比commit]

# 示例 1：生成全量报告
sh coverage_report.sh Demo.profraw

# 示例 2：生成全量 + 增量报告（对比指定 commit）
sh coverage_report.sh Demo.profraw abc1234

# 示例 3：不指定 commit 时，默认对比上一个 commit
sh coverage_report.sh Demo.profraw
```

### 8.5 手动分步生成

如果不使用一键脚本，可以手动执行每一步：

```bash
# Step 1: profraw → profdata
xcrun llvm-profdata merge -sparse Demo.profraw -o Demo.profdata

# Step 2: 查找二进制文件
MACHO=$(find ./MachOFiles -type f -exec file {} + | grep 'Mach-O' | head -n1 | cut -d: -f1)
echo "Binary: $MACHO"

# Step 3: 导出 lcov 格式
xcrun llvm-cov export "$MACHO" \
    -instr-profile=Demo.profdata \
    -format=lcov > coverage.info

# Step 4: 生成 HTML 报告
genhtml -o coverage_html coverage.info --ignore-errors category

# Step 5: 打开报告
open coverage_html/index.html
```

### 8.6 输出产物

```
.
├── Demo.profdata                  # 合并后的覆盖率数据
├── Demo.info                      # lcov 格式覆盖率数据（全量）
├── Demo_gather.info               # lcov 格式覆盖率数据（增量）
├── Demo_full_html/                # 全量 HTML 报告
│   └── index.html
└── Demo_incremental_html/         # 增量 HTML 报告（如有 diff）
    └── index.html
```

## 9. 增量覆盖率分析

### 9.1 工作原理

增量覆盖率分析只统计**本次代码变更**对应的覆盖率：

```
Git Diff → diffParser.rb → 变更行号映射 → 过滤 lcov 数据 → 增量 HTML 报告
```

### 9.2 diffParser.rb 使用

```bash
ruby gitdiff/utils/diffParser.rb \
    --diff-file=gitdiff.diff \
    --coverage-info-file=Demo.info
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `--diff-file` | ✅ | Git diff 文件路径 |
| `--coverage-info-file` | ✅ | lcov 覆盖率数据文件路径 |

**白名单文件类型：** `.m`、`.mm`、`.swift`

### 9.3 完整增量分析流程

```bash
# 1. 获取当前 commit
CURRENT=$(git rev-parse --short=7 HEAD)

# 2. 生成 Git Diff（与上一个 commit 对比）
git diff HEAD~1 HEAD --unified=0 > gitdiff.diff

# 3. 转换覆盖率数据
xcrun llvm-profdata merge -sparse Demo.profraw -o Demo.profdata

# 4. 导出 lcov
MACHO=$(find ./MachOFiles -type f -exec file {} + | grep 'Mach-O' | head -n1 | cut -d: -f1)
xcrun llvm-cov export "$MACHO" -instr-profile=Demo.profdata -format=lcov > Demo.info

# 5. 解析增量覆盖率
ruby gitdiff/utils/diffParser.rb \
    --diff-file=gitdiff.diff \
    --coverage-info-file=Demo.info

# 6. 生成增量 HTML 报告
genhtml -o incremental_html Demo_gather.info --ignore-errors category

# 7. 打开报告
open incremental_html/index.html
```

### 9.4 CI 集成示例

**GitHub Actions:**

```yaml
name: iOS Coverage Report

on:
  pull_request:
    branches: [main]

jobs:
  coverage:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0    # 需要完整 Git 历史用于 diff

      - name: Install lcov
        run: brew install lcov

      - name: Download profraw from test device
        run: |
          # 此处替换为实际的 profraw 下载逻辑
          cp /path/to/Demo.profraw .

      - name: Generate Coverage Report
        run: |
          chmod +x coverage_report.sh
          sh coverage_report.sh Demo.profraw ${{ github.event.pull_request.base.sha }}

      - name: Upload Report
        uses: actions/upload-artifact@v3
        with:
          name: coverage-report
          path: |
            *_full_html/
            *_incremental_html/
```

**Jenkins:**


## 10. 上传到覆盖率平台

### 10.1 上传覆盖率原始数据

```bash
curl -X POST https://coverage-platform.internal/api/upload/coverage \
  -F "projectId=ios-app" \
  -F "commitHash=$(git rev-parse HEAD)" \
  -F "branch=$(git branch --show-current)" \
  -F "platform=ios" \
  -F "file=@Demo.profraw" \
  -F "appVersion=1.0.0" \
  -F "deviceInfo=$(cat device.json)" \
  -H "X-API-Key: your-api-key"
```

### 10.2 上传二进制文件（可选，用于服务端解析）

```bash
# 上传 .app 包（压缩后）
zip -r Demo.app.zip Demo.app
curl -X POST https://coverage-platform.internal/api/upload/coverage \
  -F "binary=@Demo.app.zip" \
  ...

# 上传 dSYM 符号表（压缩后）
zip -r Demo.app.dSYM.zip Demo.app.dSYM
```

## 13. 常见问题

### Q1: `__llvm_profile_write_file` 返回非零值

**原因**：LLVM Profile 运行时未正确初始化或文件路径无效。

**解决**：
1. 确保在调用 `dumpCoverageData` 之前已调用 `initializeCollector`
2. 检查 Documents 目录是否有写入权限
3. 确认编译参数 `-fprofile-instr-generate` 已正确添加

### Q2: `.profraw` 文件生成为 0 字节

**原因**：App 没有执行到任何被插桩的代码。

**解决**：
1. 确保 App 有实际的操作交互
2. 检查编译参数是否正确应用到所有源文件
3. 尝试在模拟器上运行（模拟器权限更宽松）

### Q3: `llvm-cov export` 报错 "Failed to load coverage"

**原因**：二进制文件与 `.profraw` 文件不匹配（非同一构建产物）。

**解决**：
1. 确保使用**同一次构建**的 `.app` 二进制文件
2. 如果重新编译了 App，需要重新收集 `.profraw` 数据
3. 检查二进制文件是否包含调试符号（不应是 stripped 版本）

### Q4: 增量报告中没有数据

**可能原因**：

1. Git diff 中变更的文件类型不在白名单（`.m`、`.mm`、`.swift`）中
2. lcov 文件中的路径与 diff 文件路径不匹配
3. 变更的代码没有被执行到

**排查**：

```bash
# 检查 diff 文件内容
cat gitdiff.diff | grep "+++"

# 检查 lcov 文件中的源文件列表
grep "^SF:" Demo.info

# 手动运行 diffParser 查看日志
ruby gitdiff/utils/diffParser.rb \
    --diff-file=gitdiff.diff \
    --coverage-info-file=Demo.info
```

### Q5: `genhtml: command not found`

**解决**：

```bash
brew install lcov
```

### Q6: 如何支持 SwiftUI 项目？

SwiftUI 使用 `@main App` 结构体而非 `AppDelegate`，可通过以下方式集成：

```swift
import SwiftUI

@main
struct MyApp: App {
    init() {
        CoverageCollector.initialize()
    }

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

对于进入后台 / 终止的监听，使用 `ScenePhase` 环境变量：

```swift
@main
struct MyApp: App {
    @Environment(\.scenePhase) var scenePhase

    init() {
        CoverageCollector.initialize()
    }

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        .onChange(of: scenePhase) { newPhase in
            switch newPhase {
            case .background:
                CoverageCollector.dumpCoverageData()
            default:
                break
            }
        }
    }
}
```

```bash
    file1.profraw \
    file2.profraw \
```

### Q8: 真机与模拟器的区别

| 项目 | 模拟器 | 真机 |
|------|--------|------|
| 架构 | x86_64 / arm64 (Apple Silicon) | arm64 |
| 文件提取 | 直接访问文件系统 | 需通过 Xcode 或 libimobiledevice |
| 权限 | 无限制 | 沙盒限制 |
| 性能影响 | 可忽略 | 略有影响（建议仅 Debug 启用） |

### Q9: Release 包中是否包含覆盖率代码？

如果仅在 Debug 配置下添加编译参数，Release 包中**不会**包含覆盖率相关代码，不影响包体积和性能。

但 `CoverageCollector.m` 中的 `__sanitizer_cov_trace_pc_guard` 回调函数会在 Release 包中保留（因为编译器在非 Sanitizer 模式下不会生成对这些函数的调用）。如果不希望保留，可以使用 `#if DEBUG` 宏包裹。
